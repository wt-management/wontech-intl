# wontech-intl — 원텍 해외영업

이 저장소는 **`index.html` 파일 하나**가 사이트 전체다(빌드 없음, HTML 하나에 CSS·JS 전부 내장). 해외영업의 현황판·수주 칸반·딜러·활동·일정·견적 등을 관리한다.

- 라이브 주소: https://wt-management.github.io/wontech-intl/
- (구 주소 `.../wontech/intl.html` 은 이 주소로 자동 리다이렉트된다.)
- 데이터는 코드가 아니라 Supabase에 있음(아래). 이 저장소를 고쳐도 데이터는 안 지워진다.

---

## ⚠️ 반드시 지킬 규칙

1. **수정 → main에 바로 반영(직접 push).** PR 없이 main에 커밋·푸시하면 몇 분 뒤 라이브에 자동 반영된다.
   ```
   git add -A
   git commit -m "무엇을 왜 고쳤는지"
   git push
   ```
   - ⚠️ **push 전에 반드시 확인**: 브라우저에서 열어 동작 OK, 콘솔 에러 0. 바로 라이브로 나가므로 깨진 채 올리면 사이트가 깨진 채 보인다.
   - 잘못 올렸으면 즉시 되돌리기: `git revert HEAD && git push`.

2. **`index.html`만 수정한다.** `.nojekyll`(삭제 금지 — 지우면 빌드 깨짐), `.github/` 등은 건드리지 말 것.

3. **비밀키(관리자 키) 커밋 금지.** `sb_secret_...` 또는 service_role 키는 절대 넣지 말 것. 이미 들어있는 `sb_publishable_...` 키는 공개용이라 정상.

4. **공용 셸은 건드리지 말 것.** 로그인 · 권한 · 자동 로그아웃 · 사이트 이동(다른 사이트로 가는 스위처) · 비밀번호 변경 코드는 전 사이트 공통이라 오너 영역이다.

---

## 데이터 (Supabase)
- `cons_cache` 테이블에 **해외영업 전용 key**로 격리 저장된다.
- **마지막 저장이 이김:** 저장 시 블롭 전체를 덮어쓴다(동시 편집 유실 주의).
- **로컬 테스트 안전:** localhost로 열면 LOCAL 모드 = 저장 안 됨.

## 수정한 뒤 반드시 확인
- 브라우저에서 실제로 열어 동작 확인, **콘솔 에러 0개**
- 언어 토글이 안 깨지는지
- 큰 수정 후 `node --check`로 추출 스크립트 검사 권장
- 그 다음 커밋 → push

막히면 오너에게 물어보고, 큰 구조 변경은 먼저 오너 확인을 받는다.
